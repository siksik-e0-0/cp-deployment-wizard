장애 복구 시나리오

장애 시나리오 영향 범위 허용 시간 복구 후 동작
Authentik Core 전체 다운 신규 토큰 발급 차단 (OAuth2 + OIDC) JWKS 캐시 TTL (기본 1h) 자동 복구 — 토큰 재발급 재개
LDAP Outpost 전체 다운 MDS 신규 토큰 발급 차단, Principal 조회 불가 기존 MDS 토큰 TTL (기본 30m) LDAP 복구 시 MDS 자동 재연결
PostgreSQL 다운 Authentik Core 전체 불능 즉시 영향 DB 복구 후 Authentik 자동 재시작
JWKS 엔드포인트 불응답 캐시 만료 후 외부 JWT 검증 불가 JWKS 캐시 TTL (기본 1h) 엔드포인트 복구 시 자동 갱신
Redis 다운 Authentik 세션/캐시 손실, 성능 저하 세션 재생성 필요 Redis 복구 시 세션 재생성, 기능 정상

핵심 요약: 브로커 간 통신은 SCRAM-SHA-512를 사용하며 Authentik과 완전히 독립적입니다. MDS 토큰은 자체 키페어로 서명되므로, Authentik 장애 시에도 토큰 만료 전까지 CP 컴포넌트 통신은 정상 유지됩니다.

Authentik Core 1 2+ L7 (Nginx / HAProxy / Traefik) PostgreSQL + Redis (외부)
LDAP Outpost 1 2+ L4 (TCP :636) Stateless — 수평 확장 가능
PostgreSQL 1 Primary + Replica — WAL Streaming Replication
Redis 1 Sentinel 또는 Cluster — AOF / RDB Persistence

LDAP Outpost 운영 팁: Outpost는 Stateless이므로 Docker/K8s에서 자유롭게 수평 확장 가능합니다. MDS의 ldap.java.naming.provider.url에 로드밸런서 VIP를 지정하면 Outpost 장애 시 자동 페일오버됩니다. Health Check는 TCP :636 (LDAPS handshake)으로 구성합니다.
