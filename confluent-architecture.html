06 / Failure Analysis

Authentik Failure Impact

Authentik Full Outage
Affected
New client connections, C3 new logins, MDS new principal lookups
Unaffected
Broker cluster (SCRAM), existing MDS tokens valid for SR/C3
OAuth2 Provider Only
Affected
New external client token issuance blocked
Unaffected
All internal communication (MDS tokens are LDAP-based)
LDAP Outpost Only
Affected
MDS new token issuance, component re-authentication
Unaffected
Existing MDS token cache operational, Broker cluster normal
OIDC Provider Only
Affected
C3 new SSO login blocked
Unaffected
C3 -> Broker communication (existing MDS tokens), existing sessions

Key Insight: Inter-broker communication uses SCRAM-SHA-512 and is fully independent of Authentik. MDS tokens are signed with their own keypair, so CP component communication continues until token expiry even during Authentik outages.
