보안 설정

SSL/TLS 암호화 및 Listener별 인증 프로토콜을 설정합니다. RBAC 활성화 시 External/Token Listener는 OAUTHBEARER가 필수입니다.

SSL/TLS 암호화 모든 통신에 TLS 암호화를 적용합니다
상호 TLS (mTLS) 클라이언트 인증서 기반 양방향 인증
Listener 구성 ?
RBAC 연동: 이전 단계에서 RBAC이 활성화되어 External/Token Listener의 SASL이 OAUTHBEARER로 고정되었습니다.

Internal Listener Broker 간 통신 권장: PLAIN + SSL

Broker 간 내부 통신 전용. PLAIN이 표준이며, SSL로 암호화합니다. mTLS는 일반적으로 불필요합니다.

External Listener 클라이언트 통신 RBAC → OAUTHBEARER

외부 클라이언트(Producer/Consumer)가 접속하는 리스너. RBAC 사용 시 OAUTHBEARER 필수.

Token Listener 내부 서비스 통신 RBAC → OAUTHBEARER

Schema Registry, Control Center 등 내부 서비스 → Broker 통신. MDS 토큰 인증에 OAUTHBEARER 필수.
